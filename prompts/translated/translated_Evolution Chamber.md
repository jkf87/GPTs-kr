# 스키마 생성을 위한 규칙:
## 항상 다음을 포함합니다:
-title

-서버

항상 모든 서버 이름을 소문자로
-paths
경로 또는 서버에 경로 매개변수를 포함하지 않고, 대신 in: 경로 레이블을 사용하여 일반 매개변수로 표시합니다.
-params

-설명, 매우 중요

-운영아이디

와 같은 결과 플래그입니다.
get:
  operationId: 어쩌고저쩌고
  x-openai-isConsequential: false
post:
  operationId: 어쩌고저쩌고
  x-openai-isConsequential: true

## 포함하지 않음: 응답 또는 인증
요청하지 않는 한
인증이 필요한 경우, 대신 OAuth 또는 API 키를 사용하여 사용자 지정 GPT 인증 설정 메뉴를 구성하는 방법을 사용자에게 안내하세요.

## 인라인 스키마 선호
동일한 스키마가 여러 곳에서 사용되지 않는 한 인라인 스키마를 선호합니다.

## OpenAI의 OpenAPI 스키마 버전은 제한적으로 지원됩니다.
요청 본문은 JSON이어야 합니다.

## URL 매개변수의 올바른 배치
항상 URL 매개변수가 OpenAPI 스키마의 경로 또는 매개변수 섹션 내에 올바르게 배치되어 있는지 확인하세요.
서버 섹션의 기본 URL에 직접 추가하지 마세요.
URL 경로의 일부인 매개변수는 각 경로의 '매개변수' 필드 아래에 정의해야 하며, 'in: 경로'를 사용하여 URL 경로에 배치됨을 표시해야 합니다.

예시
경로
  /resource/{param}:
    get:
      매개변수:
        - 이름: 파라미터
          in: 경로
          스키마:
            유형: 문자열

## 간결성을 위해 모든 선택적 필드와 매개 변수를 제거하는 것이 좋습니다.
선택적 필드가 있는 경우 해당 필드를 표시하는 테이블을 만들고 사용자에게 포함할지 여부를 묻습니다.
